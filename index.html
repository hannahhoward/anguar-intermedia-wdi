Are you ready to build the next hot Silicon-Valley Startup: Blitter?

If there's one VC space that is wide open for almost instant success, it's social networking. You pretty much build a social networking site of any kind, and Bam! you're worth a billion dollars.

So let's do this. Your job is to build an angular site that pulls a list of 'bleets' from a backend (http://blittertime.herokuapp.com), and displays them in a list on a page. Your page should also have the ability to add new bleets, delete bleets, and 'like' a bleet. You'll need to build a custom directive to display each individual 'bleet'. And, you should be able to click on a bleet and display a detail page for just that individual bleet -- and you should use ui-router to do this.

The steps for completing your app are as follows:

  1. Setup an angular project either by hand or using a starter pack of your choice.
  2. If your seed doesn't include UI-Router, install it via npm
  3. Setup two routes with UI-Route, using $stateProvider, for the list page and the detail page of bleets. UI-Router Wiki
  4. Insert a ui-view into your main index.html.
  5. Setup a controller for each of your routes. The controller will need to load resources from the backend using $http, $resource, or Restangular. Angular Tutorial Talking to Backend, $http Documentation, $resource documentation, Restangular
  6. Setup a form and an event handler to submit new bleets. Use a POST to add the bleets to the api. Form Documentation
  7. Build a component to display individual bleets, along with the delete button and the "like" button. The directive should a link function to handle user clicking on the like button and the delete button: Where 'currentBleet' is the bleet you want to display. Component documentation
  8. Use a PUT to make a bleet become "liked" when a user clicks on a like button.
  9. Use a DELETE to remove a bleet when a user clicks on the delete button
  10. Use a ui-sref to transition to a seperate bleet detail page (which will admittedly not have any additional detail). UI-Router API Docs

The backend can be accessed on blittertime.herokuapp.com. The api documentation is as follows:
